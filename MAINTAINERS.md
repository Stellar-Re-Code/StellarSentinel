## How to Become a Maintainer

1. Consistently contribute high-quality PRs.
2. Help review other contributors' code.
3. Participate in discussions and issue triage.
4. Be nominated by an existing maintainer.

## Maintainer Responsibilities

- Review and merge pull requests within 48 hours.
- Triage new issues and assign labels.
- Keep documentation up-to-date.
- Ensure CI/CD pipelines remain healthy.
- Mentor new contributors.
